╔════════════════════════════════════════════════════════════════╗
║                  ⚠️  CRITICAL WARNING ⚠️                        ║
╠════════════════════════════════════════════════════════════════╣
║  STROBE MUST BE CONNECTED TO PIN 2 (INT0)!                    ║
║                                                                ║
║  Arduino Uno only has hardware interrupts on:                 ║
║  - Pin 2 (INT0)  ← Use this for STROBE                        ║
║  - Pin 3 (INT1)                                               ║
║                                                                ║
║  The emulator will NOT work if STROBE is on any other pin!    ║
╚════════════════════════════════════════════════════════════════╝

╔════════════════════════════════════════════════════════════════╗
║                    LOGIC LEVELS                                ║
╠════════════════════════════════════════════════════════════════╣
║  HIGH = 5V (logic 1)                                           ║
║  LOW  = 0V (logic 0)                                           ║
║                                                                ║
║  STROBE: Active LOW (low pulse indicates data ready)          ║
║  ACK:    Active LOW (low pulse = acknowledgment)              ║
║  BUSY:   Active HIGH (high level = busy)                      ║
║  SELECT: Active HIGH (high level = ready)                     ║
╚════════════════════════════════════════════════════════════════╝

╔════════════════════════════════════════════════════════════════╗
║                  PROTOCOL TIMING                               ║
╠════════════════════════════════════════════════════════════════╣
║                                                                ║
║  1. PC places data on D0-D7                                   ║
║  2. PC activates STROBE (HIGH → LOW)  ───┐                    ║
║  3. Arduino detects STROBE                │ < 2 µs (interrupt)║
║  4. Arduino activates BUSY (LOW → HIGH) ──┘                   ║
║  5. Arduino reads data                    ──┐                 ║
║  6. Arduino stores in buffer                │ ~10 µs          ║
║  7. Arduino sends ACK (LOW pulse)           │                 ║
║  8. Arduino deactivates BUSY (HIGH → LOW)───┘                 ║
║                                                                ║
║  ACK pulse width:   ~5 microseconds                           ║
║  BUSY time:         ~10 microseconds                          ║
║  Maximum rate:      ~100 kB/s (theoretical)                   ║
╚════════════════════════════════════════════════════════════════╝

╔════════════════════════════════════════════════════════════════╗
║                   IMPORTANT NOTES                              ║
╠════════════════════════════════════════════════════════════════╣
║                                                                ║
║  • Use short cables (< 2 meters) to avoid noise              ║
║  • Connect ALL GND pins (18-25) to Arduino GND               ║
║  • Arduino Uno uses 5V TTL levels (compatible with LPT)      ║
║  • Pin D10 has external interrupt (essential for STROBE)     ║
║  • Pin D13 has onboard LED (useful for debugging)            ║
║  • Serial rate: 115200 baud (maximum speed)                  ║
║  • Buffer: 256 bytes circular                                 ║
║                                                                ║
